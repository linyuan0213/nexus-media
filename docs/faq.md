# 常见问题

## 安装部署

### Q: Docker 启动后无法访问前端？
检查端口映射：
```bash
docker compose ps
```
前端默认映射 `3000:8080`，后端 `3001:3000`。访问 http://localhost:3000。

### Q: 首次登录失败？
默认账号：
- 用户名: `admin`
- 密码: `password`

若修改过密码但遗忘，需删除 `config/user.db` 重置（会丢失用户数据）。

### Q: 配置文件修改后未生效？
后端使用 `watchdog` 监控配置文件变更，但部分配置需重启服务生效。建议修改后观察日志或重启容器。

## 媒体识别

### Q: 无法识别媒体？
1. 检查 **设置 > 基础设置 > 媒体** 中 TMDB API Key 是否配置
2. 检查网络是否能访问 `api.themoviedb.org`
3. 尝试开启 **实验室 > 辅助识别** 或 **WEB增强识别**

### Q: 电视剧集数识别错乱？
开启 **设置 > 基础设置 > 媒体 > 启用集数映射**，可在媒体详情页手动调整集数映射关系。

## 下载与转移

### Q: 下载完成不自动转移？
1. 下载器配置中是否开启「监控」
2. 检查下载器设置的「保存目录」与「媒体库目录」是否在同一个磁盘（硬链接需求）
3. 查看日志中是否有转移失败的错误信息

### Q: 硬链接失败？
硬链接要求源目录和目的目录在同一磁盘分区。Docker 环境下需映射上级目录：
```yaml
volumes:
  - /data:/data  # 统一定上级目录
```

### Q: 目录同步不触发？
1. 检查宿主机 inotify 限制：
   ```bash
   echo fs.inotify.max_user_watches=5242880 | sudo tee -a /etc/sysctl.conf
   sudo sysctl -p
   ```
2. 网盘/SMB 不支持实时通知，需开启目录同步的「兼容模式」或使用「下载器监控」

## 站点

### Q: 站点无法连通？
1. 检查 Cookie 是否有效（过期需更新）
2. 检查代理设置是否正确
3. 使用「站点连通性测试」排查具体错误

### Q: 馒头(M-Team)怎么配置？
在站点维护中添加请求头：
```json
{"x-api-key": "你的令牌"}
```
令牌在馒头控制台 > 实验室 > 存取令牌获取。

### Q: 肉丝(Rousi)怎么配置？
在站点维护中添加请求头：
```json
{"Authorization": "Bearer 你的Passkey"}
```
Passkey 在肉丝账户设置中复制。

## 刷流

### Q: 刷流任务下载了非免费种子？
检查任务配置中的「促销」选项：
- 选择「全部」不过滤促销类型
- 状态图标为绿色时才表示仅下载 FREE 种子

### Q: 刷流任务添加后没反应？
1. 检查任务状态是否为「运行中」
2. 检查执行周期是否正确（支持 cron 表达式）
3. 查看任务详情中的「种子列表」确认是否有下载记录

## 通知

### Q: Telegram 通知收不到？
1. 检查 Telegram Bot Token 和 Chat ID
2. 如果使用 Webhook，确保外网可访问并配置正确的域名
3. 检查 **安全设置** 中是否允许 Telegram IP 来源

### Q: 消息模板不生效？
1. 检查 JSON 格式是否正确
2. 确认消息类型名称拼写正确
3. 查看系统日志是否有模板渲染错误

## 性能

### Q: 内存占用过高？
1. 降低 RSS 订阅频率
2. 关闭不必要的站点数据统计
3. 减少同时运行的刷流任务数量

### Q: 磁盘空间不足？
刷流任务中可配置「磁盘剩余空间」删种规则，低于阈值时自动删除种子释放空间。

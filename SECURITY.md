# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 最新 release (v3.x) | 接受安全更新 |
| dev 分支 | 接受安全更新 |
| v2.x 及以下 | 已停止维护 |

## 报告漏洞

如果你发现了安全漏洞，请通过以下方式报告：

1. **私下报告**：不要通过公开 issue 报告安全问题。请发送邮件至项目维护者。
2. **提供详情**：请提供复现步骤、受影响版本、以及可能的修复建议。
3. **响应时间**：我们会在 48 小时内确认收到报告，并在 7 天内给出初步评估。

## 安全实践

- 定期更新依赖：`uv sync`
- 不要在生产环境使用默认密码
- 建议配置 HTTPS 反向代理
- Docker 部署时使用非 root 用户
